Phase 1 — Stability Fixes

These were bugs making existing features silently fail. All fixed and pushed to main.

✓
Vite proxy for local dev
Search, synthesis, and "what do you see?" were returning 404 in local dev because Vite had no way to reach the API routes. Now /api/* proxies to Vercel dev.
vite.config.js
✓
Fixed the RPC fallback bug
When the main Supabase query failed and the RPC fallback succeeded, the code was still reading the failed response. Now it correctly tracks which response to use.
api/gemini.js
✓
Search no longer fires multiple times
Every view switch was adding duplicate click handlers. After 5 switches, one search fired 5 times. Now old handlers are cleaned up automatically with AbortControllers.
hunter/src/main.js
✓
Project loading shows errors now
If Supabase fails when opening a project, you now see a red toast instead of the UI silently breaking.
hunter/src/main.js
✓
Missing CSS variables defined
--np-radius-lg and --np-duration-md were used but never defined. Now they actually do something.
hunter/src/style.css
✓
Escape key closes the new-project modal
Was checking for a DOM ID that didn't exist. Now finds the modal by its class name.
hunter/src/main.js
✓
Demo mode doesn't hijack real projects
Creating a new project with zero sources was triggering demo mode. Now demo only activates when the database isn't configured at all.
